01The Gap

What every new project costs today?

The first week of any project is the same week, over and over.

Every project starts with the same work: repository scaffolding, a project plan, technology decisions, initial wireframes, and the checklists that somehow never existed the last time.

It's repetitive, inconsistent, and almost entirely manual. Documentation drifts out of sync, every team develops its own process, and your most experienced engineers spend valuable hours on setup instead of the decisions that actually move the project forward.

What the assistant does instead?

It absorbs the repeatable part, then hands the judgment back.

One brief goes in. Agents split the work, produce a first-draft project package, and run a quality pass on their own output before anyone looks at it.

Nothing ships on its own. Every run ends at a human review gate. The machine gets you to the 80%; the person decides on the 20% that matters.

02One Run, Seven Agents

The pipeline is a relay, not a black box.

06Why this stays Gated

The reasons it works are the reasons to be careful.

An agent that can read your files and touch your machine is useful for exactly the reason it's dangerous. These are documented, not hypothetical, and they're why every run stops at a human.

HIGH

Prompt Injection

Instructions hidden in a file, a webpage, or a repo the research agent reads can hijack the run, steering it to do something the brief never asked for.

HIGH

Host-level Tool Access

The runtime can run commands on the host. That reach is what lets it scaffold a repo, and what makes a bad or hijacked step have real consequences.

MED

Unvetted Third-Party Skills

Community skills and plugins run with the agent's permissions. An unreviewed one is untrusted code executing inside your environment.

MED

Hallucination / Accuracy

Output can be confidently wrong. Every deliverable is a draft a human validates; nothing is trusted just because it looks finished.

MED

Local Hardware Cost

Kimi K2.6 is ~1T parameters; self-hosting is hardware-heavy, so the pilot uses the API rather than standing up its own cluster.

MED

Integration Effort

Wiring OpenClaw into real internal tools takes setup, testing, and maintenance. It isn't a drop-in the day it lands.

MED

Reliability

Long autonomous runs can drift or stall. Human checkpoints keep failures cheap and visible instead of silent and expensive.

07The Verdict

Run it as a limited internal pilot: earn the expansion, don't assume it.

01

Pilot on non-confidential projects only.

02

Measure two things: accuracy and time saved.

03

Humans validate every output. No exceptions during the pilot.

04

Expand department by department, only after a security review clears it.

This is the same shape as everything else Elchai ships: prove it before you expand it. The gate stays until the evidence says it can move.
